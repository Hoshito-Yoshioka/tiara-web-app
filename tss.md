# tech-stack-selection.md

## 1. プロジェクト方針

本プロジェクト「tiara-web-app」は、技術習得（モダンスタックの理解）と、バーとしての高級感・信頼性を提供することを目的とする。

## 2. フロントエンド (Frontend)

| 技術                 | 選定結果           | 選定理由・経緯                                                                                     |
| :------------------- | :----------------- | :------------------------------------------------------------------------------------------------- |
| **Framework**        | **Vue 3 (Vite)**   | Composition APIによる柔軟な設計と、学習のしやすさを両立。Viteによる高速な開発体験を優先。          |
| **Language**         | **TypeScript**     | 実行時エラーの防止と、エディタの補完機能を最大限活かすための必須選択。                             |
| **Styling**          | **Tailwind CSS**   | ユーティリティファーストによる迅速なUI構築と、デザインの一貫性保持のため。                         |
| **UI Library**       | **shadcn-vue**     | コンポーネントのコードを直接管理できるため、カスタマイズ性が高く、UIの仕組みを学ぶのに適している。 |
| **State Management** | **Pinia**          | Vue 3標準の型安全な状態管理。Vuexよりもシンプルで直感的なため採用。                                |
| **Animation**        | **@vueuse/motion** | Vueのライフサイクルに密着した直感的な記述が可能。ミニマルかつリッチな演出を実現するため。          |

## 3. BFF (Backend For Frontend)

| 技術          | 選定結果    | 選定理由・経緯                                                                                    |
| :------------ | :---------- | :------------------------------------------------------------------------------------------------ |
| **Runtime**   | **Node.js** | Docker環境での構築が標準的であり、ライブラリの制限（Workers制限等）を気にせず開発に集中するため。 |
| **Framework** | **Hono**    | 極めて軽量かつ高速。TypeScriptでの型安全なバリデーション（Zod等）とルーティングが強力なため。     |

## 4. バックエンド (Core API)

| 技術          | 選定結果   | 選定理由・経緯                                                                                     |
| :------------ | :--------- | :------------------------------------------------------------------------------------------------- |
| **Language**  | **Golang** | 高い実行パフォーマンスと、シンプルで厳格な言語仕様による保守性の高さを評価。                       |
| **Framework** | **Echo**   | Honoの設計思想に近い「クリーンで読みやすい」コードが書ける。ミドルウェア管理のしやすさも選定理由。 |

## 5. データベース (Database)

| 技術         | 選定結果       | 選定理由・経緯                                                                                  |
| :----------- | :------------- | :---------------------------------------------------------------------------------------------- |
| **RDBMS**    | **PostgreSQL** | 複雑なクエリやデータ整合性に強く、JSONB型などの柔軟な拡張性も備えているため。                   |
| **SQL Tool** | **sqlc**       | SQLから型安全なGoコードを自動生成する。ORMの抽象化を避け、SQLを学びつつ型安全性を確保するため。 |
| **Driver**   | **pgx**        | PostgreSQL用の最も高性能かつ標準的なGoドライバー。                                              |

## 6. インフラ・共通ツール (Infra / Common)

| 技術            | 選定結果     | 選定理由・経緯                                                                               |
| :-------------- | :----------- | :------------------------------------------------------------------------------------------- |
| **Container**   | **Docker**   | 開発環境の差異をなくし、PostgreSQLなどの依存関係をクリーンに保つため。                       |
| **Pkg Manager** | **pnpm**     | 高速なインストールと、モノレポ構成（frontend/bff/backend）に適したワークスペース機能のため。 |
| **Formatter**   | **Prettier** | コードスタイルの議論を避け、自動整形による可読性維持のため。                                 |
